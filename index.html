PROJECT IDEA

The project idea is a fingerprint-based ATM system, where instead of using a card to access your account the user will only need to use their own fingerprint to gain authorization and be able to conduct transactions, withdrawals, etc. This application will store the user’s fingerprint to their corresponding card in a database making it much simpler and less stressful if in the worst-case scenario, you do lose your card all the user will need to provide their fingerprint.
Owning and carrying a credit/bank card is a must need for those transitioning to adulthood, but to abruptly lose this vital item important for daily life can potentially hinder their completion of daily tasks. This project will prove useful as the most common type of identity theft reported in 2019 is credit card theft. By implementing this new fingerprint-based system using python in conjunction with MySQL to store fingerprints this will be a cheap solution to the ongoing problem of identity theft.

MAIN FEATURES

The main feature of this project is when users are creating a new or existing credit/debit card account they would be informed to assign their fingerprint to the corresponding account(s) using a fingerprint scanner. The fingerprint with the assigned account would then be transferred into a database containing all the user’s potential data like what type of card, current savings, name, card number, personal email etc. The database filled with accounts will then be connected with all the current and future ATMs via a server hosted by the company which will receive live changes whenever a change is made to any account. For a user to access their account they would first need to go to a local atm owned by the company they would then to scan their fingerprint and for extra safety a 4-digit pin, this will give the user authorisation and send them to the card selection page which they can select their desired card if their fingerprint is bound to multiple accounts. When the desired card is selected, they will then be sent to the options page where there will be several classic features like withdrawal, deposit balance inquiry, print statement, payments etc, but there will also be two new features added like print statement to email, varying note denominations. The print statement to email simple means that when choosing the option to print a statement the user can either choose to print a receipt or directly email it to their account’s set email address. The varying note denominations means that instead of only printing out 50-dollar notes for withdrawals over $100 the user can choose which note denomination they would want to print out, however this option will be capped to stop users from withdrawing fifty, five-dollar notes for a total of $250.
To run this application the given computer will need to have any python code editor program installed and the database management system MySQL installed as well to contain the data. The company investing in this project will also need a server to store, send and receive data, preferable on that matches the number of users they currently have. On top of that atm’s may need to be replaced or modified to suit the new needs of the project.
The main software will need to be written by an experienced programmer as they are dealing with user’s money and there will be severe repercussions if there are any mistakes present in the code. To find the needed programmer several job advertisements can be set up to seek the suited person(s).
If the project is successful the finger-print atm feature will gradually gain more recognition and become more mainstream and with that see the general decline of people carrying around bank cards and therefore, see the overall decline of identity theft in the form of credit card fraud.
